# Novasphere — GitHub Release Checklist

Use this checklist before cutting a GitHub release and publishing packages to npm.

---

## 1. Version alignment

- [ ] **All publishable packages on target version (e.g. 0.1.0)**  
  - Current: 9 packages at `0.0.1` (tokens, agent-core, tenant-core, ui-glass, ui-bento, ui-charts, ui-agent, ui-auth, ui-shell).  
  - To release as 0.1.0: run `pnpm changeset version` after adding a changeset that bumps all packages to 0.1.0, or bump `version` in each package.json and run `pnpm install`.

---

## 2. CI green on `main`

- [ ] **Full CI sequence passes on the release branch (e.g. `main`):**
  ```bash
  pnpm -r typecheck   # Zero errors
  pnpm -r lint        # Zero warnings (ESLINT_USE_FLAT_CONFIG=false is set by root script)
  pnpm -r test        # All tests green
  pnpm -r build       # All packages and apps/demo build
  ```

---

## 3. Package build artifacts

- [ ] **Each package has correct `dist/` output:**
  - `index.js` (ESM) exists
  - `index.cjs` (CJS) exists  
  - `index.d.ts` (types) exists
  - No `src/` or raw `.ts` source files in `dist/` (only compiled output and `.d.ts`)

---

## 4. Publish dry-run

- [ ] **Changeset publish dry-run succeeds (fails only on npm auth in CI):**
  ```bash
  pnpm changeset publish --dry-run
  ```
  - Expect: 9 packages would publish (@novasphere/tokens, agent-core, tenant-core, ui-glass, ui-bento, ui-charts, ui-agent, ui-auth, ui-shell).
  - In a logged-out or CI environment, auth errors (ENEEDAUTH) are expected; the dry-run still confirms which packages and versions would be published.

---

## 5. Demo app deploy

- [ ] **Demo URL is live and working:**
  ```bash
  vercel --prod --prebuilt
  # or: vercel deploy
  ```
  - Confirm the deployed demo URL loads and the dashboard renders (e.g. Bento grid, shell, no runtime errors).

---

## 6. Documentation and links

- [ ] **README has correct links:** repo URL, demo URL, package docs / Storybook links.
- [ ] **CONTRIBUTING.md** is up to date (setup, changesets, branching).

---

## 7. Storybook

- [ ] **All stories render:** run Storybook per package (or root) and confirm no broken stories or missing dependencies.

---

## 8. Ollama quick start (optional but recommended)

- [ ] **Ollama quick start tested on a clean machine:**  
  - Clone, `pnpm install`, `pnpm dev`, open app, confirm Copilot works with local Ollama (or falls back to Mock/WebLLM as documented).

---

## Post-checklist: cutting the release

1. Merge all release-prep PRs to `main`.
2. Run `pnpm changeset version` (if using changesets to bump versions).
3. Run `pnpm changeset publish` (requires npm login and 2FA if enabled).
4. Create a GitHub release with tag and release notes; attach CHANGELOG section if generated by changesets.
